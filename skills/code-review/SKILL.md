# Code Review Skill

## Workflow

### 1. 确定审查范围

- 使用 Read 工具读取目标文件
- 如果用户指定了具体文件，审查这些文件
- 如果用户未指定，使用 Glob 查找最近修改的文件
- 使用 Grep 搜索潜在的代码异味（如 TODO、FIXME、HACK）

### 2. 执行审查（按以下维度逐一检查）

**代码质量**
- 命名是否清晰、一致
- 函数是否单一职责
- 是否有重复代码
- 注释是否必要且准确

**安全性**
- 是否有 SQL 注入、XSS 等漏洞
- 敏感信息是否硬编码
- 输入验证是否充分
- 权限检查是否正确

**性能**
- 是否有不必要的循环或重复计算
- 数据库查询是否高效
- 资源是否正确释放

**可维护性**
- 错误处理是否完善
- 是否有充分的类型注解
- 模块耦合度是否合理

### 3. 生成审查报告

输出格式：
```
## Code Review: [文件/PR名称]

### 概述
- 审查文件数: N
- 严重问题: N
- 建议改进: N

### 严重问题
1. [问题描述] - [文件:行号]
   - 风险: [说明]
   - 建议: [修复方案]

### 建议改进
1. [建议描述] - [文件:行号]
   - 原因: [说明]
   - 建议: [改进方案]

### 总结
[整体评价和建议]
```

## 自由度

中等。审查维度是固定的，但具体建议可以灵活表达。
